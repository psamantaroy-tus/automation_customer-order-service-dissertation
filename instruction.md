CHAPTER 1:	INTRODUCTION
Software systems are increasingly used to support business, educational, industrial, and organisational processes. They help users automate repetitive tasks, manage data, improve communication, and support decision-making. As digital transformation continues to expand, there is a growing need for software solutions that are reliable, secure, usable, and scalable.
However, many existing approaches to [insert problem/domain] still rely on manual processes, disconnected tools, spreadsheets, emails, or outdated software systems. These approaches can lead to inefficiency, duplication of work, data inconsistency, poor user experience, and limited ability to monitor or analyse information effectively.
This thesis focuses on the design, development, and evaluation of [insert system name], a software-based system intended to address the limitations of current approaches in [insert problem/domain]. The project applies software engineering principles to identify requirements, design a suitable architecture, implement a working prototype, and evaluate the system through testing and analysis.
1.1	Motivations and Research Question
The motivation for this research arises from the need to improve the way [insert process/problem] is currently managed. In many cases, users depend on manual methods or separate software tools to complete tasks. This can result in delays, errors, duplicated work, poor communication, and difficulty accessing important information.
From a software engineering perspective, this creates an opportunity to develop an integrated system that centralises data, automates key tasks, and provides users with a more efficient and user-friendly way to complete their work. A well-designed software system can reduce manual effort, improve data accuracy, support better decision-making, and increase overall productivity.
Existing systems related to [insert topic] provide some useful features, but they may be limited in terms of customisation, usability, cost, scalability, security, or suitability for the specific needs of the target users. Therefore, this research investigates the development of a tailored software solution that addresses these limitations.
The main research question addressed in this thesis is:
How can a software-based system be designed, implemented, and evaluated to improve the efficiency, usability, and reliability of [insert problem/domain]?

The following research sub-questions help to address the main research question, which are as follows:
subRQ 1: What are the limitations of existing systems or approaches used for [insert problem/domain]?
subRQ 2: What functional and non-functional requirements should the proposed system include?
subRQ 3: Which software architecture, tools, and technologies are suitable for the development of the proposed system?
subRQ 4: How effective is the developed system when evaluated through functional, usability, and performance testing?
Sub-research question 1 is addressed in the first experimental evaluation described in Chapter 4. Sub-research questions 2, 3 and 4 are addressed in Chapter 5 which presents the results of the second experimental evaluation.
1.1.1	Research Aims and Objectives
The aim of this research is to design, develop, and evaluate a software-based system that addresses the identified limitations in current approaches to [insert problem/domain]. The objectives of this research are:
1.	To review existing literature, systems, and technologies related to [insert topic].
2.	To identify the main limitations of existing approaches used in [insert problem/domain].
3.	To define the functional and non-functional requirements of the proposed software system.
4.	To design the system architecture, database structure, user interface, and main workflows.
5.	To implement a working software prototype using appropriate programming tools and frameworks.
6.	To test the system using functional, integration, usability, and performance testing methods.
7.	To evaluate whether the developed system addresses the research question and meets the project objectives.
8.	To identify the limitations of the research and propose future improvements.

This thesis makes several contributions to the area of [insert topic/domain]. First, it provides a review of existing literature, systems, and technologies relevant to the research problem.
